Ajax的原理及其实现

这里通过一些简单的项目来解释一下Ajax的原理，当然主要的工作是依靠后端来完成，但也需要你有前端的一些基础，否则实操起来会比较困难。

Ajax的原理

Ajax简介

Ajax = Asynchronous JavaScript and XML（异步的 JavaScript 和 XML）,通过在后台与服务器进行少量数据交换，AJAX 可以使网页实现异步更新。这意味着可以在不重新加载整个网页的情况下，对网页的某部分进行更新。

  • Ajax 不是新的编程语言，而是一种使用现有标准的新方法。
  • Ajax 是与服务器交换数据并更新部分网页的艺术，在不重新加载整个页面的情况下。

如上图所示，是Ajax和传统的Web应用模型工作方式的比较。传统的Web工作模式非常简单，就是浏览器发送HTTP请求，服务器接受之后进行处理，最后返回一个HTML+CSS页面显示在客户端，这种工作方式难免会出现用户等待的情况，因此Ajax也就应运而生了。

与传统的Web应用模型相比，Ajax在浏览器与Web服务器之间新增加了一层引擎，这个引擎是浏览器在第一次与服务器进行通信时就下载到客户端的浏览器中，直到客户端与服务器通信完毕。当用户与服务器产生会话时，Ajax直到如何响应用户的交互请求，同时能够处理一定的逻辑，不仅如此Ajax给服务器传递的并不是整个页面，只是需要处理的数据，这就大大减轻了服务器的压力，使得页面的模板、布局等能够得到保持。

Ajax原理

XMLHttpRequest API是Ajax的核心。

XMLHttpRequest(XHR)对象用于与服务器交互，通过 XMLHttpRequest 可以在不刷新页面的情况下请求特定 URL，获取数据。这使得网页能够在不影响用户操作的情况下，更新页面的局部内容。

Ajax的实现

Ajax的实现步骤：

  1. 创建XMLHttpRequest对象，也即一个异步调用对象；
  2. 创建一个新的HTTP请求，并指定该HTTP请求的方法、URL以及验证信息；
  3. 设置响应HTTP请求状态变化的函数；
  4. 发送HTTP请求；
  5. 获取返回的数据；
  6. 实现局部刷新。
