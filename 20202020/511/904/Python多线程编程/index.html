Python多线程编程

虽然题目是多线程编程，但在Python中并不推荐你通过多线程编程实现多任务,而是通过多进程的方式实现多任务，这是因为Python中的多线程无法调用使用多核资源，如果你的程序只是频繁使用I/O资源，那建议使用多线程即可，若需要频繁使用CPU资源，则建议使用多进程。

多任务

我们所使用的操作系统能够同时运行多个任务，例如：可以一边使用浏览器上网，一边通过音乐播放器听音乐，这也就是操作系统的多任务。多任务能够同一时间执行多个任务，充分利用CPU效率，提高程序的执行效率。

多任务的实现可以通过并发和并行两种方式实现。

  • 并发：在一段时间交替去执行多个任务。（业务数量大于CPU的核心数）
  • 并行：在一段时间真正的同时一起执行多个任务。（业务数量小于CPU的核心数）

Python中我们可以使用进程和线程实现多任务。

进程

进程（Process）是资源分配的最小单位，它是操作系统进行资源分配和调度运行的基本单位，通俗理解为一个正在运行的程序就是一个进程。例如你电脑上正在运行的QQ、Pycharm等其他软件，它们都是一个进程。

线程

线程是程序执行的最小单位，实际上进程只负责分配资源，而利用这些资源执行程序的是线程，也就是说进程是线程的容器。一个进程中最少有一个线程来执行程序，同时线程自己不拥有系统资源，只需要一点在运行中必不可少的资源，但他可以和同属一个进程的其他线程共享全部的资源。

线程和进程的对比。

  1. 关系对比

    ​ a. 线程是依附再进程中的，没有进程就没有线程。

    ​ b.一个进程默认提供一条线程，进程可以创建多条线程。

  2. 优缺点对比

    ​ a.进程可以使用多个内核，但会带来大的开销。

    ​ b.线程资源开销小，但不能使用多核。

  3. 区别对比

    ​ a.创建进程的资源开销比创建线程的开销大。

    ​ b.进程是系统资源分配的基本单位，线程是CPU调度的基本单位。

    ​ c.线程不能够独立执行，必须依存在进程中。
